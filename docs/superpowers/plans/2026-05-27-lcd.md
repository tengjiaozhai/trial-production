# LCD 管控物料表解析与单元格双列展示 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 从上传的“管控物料表”Excel中解析 LCD 对应的一供/二供编码与供应商，并在 Step2 自动填充 `lcd`，同时只在 `LCD` 这一行把单元格拆成 1 或 2 个小列显示，消除当前 `A1-主供 缺少值` 的 LCD 冲突。

**Architecture:** 上传时立即解析“配置表”和“管控物料表”，状态里只保存派生结果，不保存原始 `File`。`PCBA配置表` 继续负责 `pcba/band/storage`，并补充 `projectName`；“管控物料表”负责按 sheet 提取 LCD 的一供/二供候选。`startAutoCalc` 按 `PcbaOption.projectName -> 可见业务 sheet` 解析 LCD 文本，`TrialProductionTable` 只在 `lcd` 行内做双列展示，`sku.supplies` 结构不扩散到其他字段。

**Tech Stack:** TypeScript, React 19, Vitest, xlsx.

---

## Reference Notes

- 当前真实文件 `../X6728传音管控物料表_V3.5-2025-10-10.xlsx` 的业务表头在第 `4` 行，首条数据在第 `5` 行；不能把第 `1` 行当表头。
- 当前 workbook 中 `X6728` 是可见业务 sheet，`X6728B` 和 `删除` 是隐藏 sheet；默认只解析可见业务 sheet。
- 当前 LCD 在物料名称列里写的是 `LCM`，不是 `LCD`；归一化后 `LCD / LCM / 显示屏 -> LCD`。
- 当前“编码”列名是 `传音编码`，实现必须用 `/编码/` 模糊匹配，不写死完整列名。
- 当前“一/二供”列名是带换行的 `一/\n二供`，实现必须先做去空白/去换行归一化后再匹配。
- 展示形态已锁定为“单行拼接”：每个 LCD 小列显示 `17401942 一供 天马` 这种一行文本。
- 继续沿用 `troubleshooting.md` 的 fail-closed 原则：解析不到可靠值时留空并保留冲突，不允许从错误 sheet 或错误表头猜值。

- [ ] **Step 3: 在 `src/lib/utils.ts` 新增 LCD 解析工具**

```ts
export function normalizeMaterialName(raw: string): string {
  const compact = String(raw ?? '').replace(/\s+/g, '').replace(/[()（）_/-]/g, '').trim().toUpperCase();
  if (compact === 'LCD' || compact === 'LCM' || compact === '显示屏') return 'LCD';
  return compact;
}

export async function extractManagedMaterialWorkbook(file: File): Promise<ManagedMaterialWorkbook> {
  // 1. 只遍历 visible sheet
  // 2. 在前 10 行里找同时包含 “物料名称 / 编码 / 供应商 / 一二供” 的真实表头
  // 3. 从表头下一行开始扫描
  // 4. 物料名称归一化后等于 LCD 的行才保留
  // 5. 只保留 supply 为 一供/二供 的行
  // 6. 每条记录输出 { supply, code, vendor, text }
}

export function resolveLcdOptionsForProject(
  projectName: string,
  workbook?: ManagedMaterialWorkbook
): LcdSupplyOption[] {
  return projectName && workbook?.lcdBySheet[projectName]
    ? workbook.lcdBySheet[projectName]
    : [];
}

export function serializeLcdOptions(options: LcdSupplyOption[]): string {
  return options.map(option => option.text).join(' / ');
}
```

实现约束：
- 表头查找不能写死第 1 行，也不要写死第 5 行；当前真实文件是第 4 行表头、第 5 行数据。
- “编码”列用 `/编码/` 匹配；“一/二供”列先做去空白、去换行归一化再匹配。
- 同一 sheet 内 LCD 如果出现重复的一供或二供，只保留第一条，顺序固定 `一供 -> 二供`。
- 隐藏 sheet 直接忽略；找不到同名 visible sheet 时返回空数组，不能从其他 sheet 猜值。

## Acceptance & Assumptions

- 只解析文件名包含 `管控物料表` 的 workbook。
- 只解析可见业务 sheet；当前真实 workbook 中 `X6728B`、`删除` 为隐藏 sheet，默认不参与。
- sheet 选择按 `PcbaOption.projectName` 精确匹配可见 sheet 名；匹配失败时 fail-closed，保留 LCD 冲突。
- `LCD / LCM / 显示屏` 只做归一化等值匹配，不做模糊子串匹配。
- 只采集 `一供`、`二供`；`三供` 和其他值忽略。
- LCD 单元格内部最多显示两列，顺序固定 `一供 -> 二供`。
- 不新增原始 `File` 到 React 状态或历史记录，避免破坏 `localStorage` 序列化路径。
